1. ДОБАВИТЬ ВЕРШИНУ: Считывается id вершины из соответствующего поля и имя вершины, в случае, если имя "пустое", в качестве имени будет присвоен id. Id - только цифры, иное ввести и не получится :3 Также при создании вершины к введенному вами имени будет добавлен и id (для удобства работы с матрицей смежности).
  2. ДОБАВИТЬ ДУГУ: Считываются поля FROM и TO (принимаются только цифры), соответсвенно id вершин, между которыми нужно образовать связь. Ввод id несуществующей дуги будет проигнорирован. Также считывается поле Weight - вес предполагаемой дуги. Из одной вершины в другую может быть несколько дуг с разными весами. Если вы оставите поле Weight пустым, дуга будет создана и будет числиться, но в алгоритме Дейкстры она засчитана не будет.
  3. УДАЛИТЬ ВЕРШИНУ: в поле id введите id удаляемой вершины, ввод имени не обязателен.
  4. УДАЛИТЬ ДУГУ: в поля FROM TO введите id соответствующих концов. Если дуг подходящих по описание несколько, будут удалены все. Введенный вес игнорируется.
  5. ОЧИСТИТЬ: безвозвратно удалит все введенное и созданное в процессе игры в граф.
  6. ДЕЙКСТРА: из поля id считыается id вершины, для которой считаем все кратчацйшие пути, если там пусто, то по дефолту будет посчитано от первой в списке вершин вершины. В результате работы будет выведена табличка кратчайших путей.
  7. ДЕФОЛТНЫЙ ГРАФ: будет сгенерирован граф с 5 вершинами. Его можно менять как угодно, он существует для удобства тестирования.
  8. ВИЗУАЛИЗАЦИЯ: можно двигать, масштабировать, выделять вершину и дуги, "трясти" граф, передвигать мышкой.
  9. МАТРИЦА СМЕЖНОСТИ: если пути нет или Weight между ними пустой - infinity, пути нет, если есть, то соответствующий дуге вес.
